# SDCC 8051 Project Template (CMake + SDCC)

This repository provides a CMake-based build setup for developing **8051 microcontrollers** using the **SDCC** compiler, along with an optional configuration file for CLion's custom editor settings. Proudly generated by DeepSeek and tested/modified on Windows 11 25H2 + CLion 2026.1.

## File Overview

| File | Description |
|------|-------------|
| `CMakeLists.txt` | Generic CMake build script. Defines the executable, include directories, and compile options (`--opt-code-size`). |
| `sdcc_toolchain.cmake` | Generic CMake toolchain file for SDCC cross-compilation. Specifies compiler path, output format (Intel HEX), compile/link rules, and flags. |
| `SDCC_for_8051.yaml` | **CLion-specific** custom editor settings file. Helps CLion's code insight engine understand SDCC's predefined macros and include paths. Not required for command-line builds. |

## Prerequisites

- [SDCC](http://sdcc.sourceforge.net/) installed and available in `PATH`.
- [CMake](https://cmake.org/) (version 3.20 or higher).
- (Optional) [CLion](https://www.jetbrains.com/clion/) with custom environment support.

## Usage

### Generic Build (Command Line)

```bash
mkdir build && cd build
cmake .. -DCMAKE_TOOLCHAIN_FILE=../sdcc_toolchain.cmake
cmake --build .
```

The output will be a `.hex` file ready for flashing.

### Using with CLion

1. Place `SDCC_for_8051.yaml` in your CLion project root (or in `/.idea/`).
2. In CLion, go to **Settings → Editor → Code Insight → C/C++ → Custom Compiler Configuration**, and select this YAML file.
3. Use the same CMake toolchain file (`sdcc_toolchain.cmake`) in **Settings → Build, Execution, Deployment → CMake** (add `-DCMAKE_TOOLCHAIN_FILE=../sdcc_toolchain.cmake` to CMake options).

CLion will then correctly parse SDCC-specific macros and provide proper code assistance.

## Notes

- The `sdcc_toolchain.cmake` forces a static library for CMake's test builds to avoid linker issues.
- The final `.hex` file is generated by `packihx` from the intermediate `.ihx` file.
- You can pass additional compiler flags by setting `SDCC_EXTRA_FLAGS` before running CMake.
